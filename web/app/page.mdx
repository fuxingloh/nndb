A first-principles study of **how many top-k vector queries per second a single CPU box can really do** over modern embeddings — and what actually moves that number.

It works through the techniques behind state-of-the-art vector search — binary quantization, learned rotations, product quantization, SIMD scan kernels — building up from a brute-force baseline, measured the [ANN-Benchmarks](http://ann-benchmarks.com/) way (recall / QPS / latency), keeping only what the data justified. Sixty-one numbered experiments later, this is the engine that came out best and the lessons that got there. The full lab notebook is at [/experiments](/experiments).

## The wall is memory, not compute

Brute force — distance to all 1M vectors per query — does ~9 QPS. The instinct is to speed up the kernel (FMA, AVX-512, batching); I did, and it roughly doubled, then stopped. The bottleneck was never compute. It's **memory bandwidth**: a query scans the whole base, and at 4 KB/vector that's 4 GB moved from RAM every time. You can't out-compute the bus.

That reframes everything into two levers: **send fewer bytes**, or **spend fewer/cheaper instructions** on each byte. The rest of the project is those two. ([003–008](/experiments/004-cascadelake-avx512))

## The funnel

Fewest bytes: keep one **sign bit per dimension** — 32× smaller. Compare two of these codes with Hamming distance (XOR, count differing bits). It's coarse, so use it as a **funnel**: scan all N with the 1-bit code for a few-hundred shortlist, then re-rank only those with the real vectors. Recall ~0.99, QPS 9 → ~500 — the biggest single step. ([009](/experiments/009-binary-rerank))

## Lever two: the instruction matters more than the math

Counting set bits is `VPOPCNTDQ` — one SIMD instruction over eight 64-bit words. A "smarter" scheme with better recall-per-bit (asymmetric scoring via a lookup table) *lost by ~30×*, because a table lookup is a **gather** the CPU can't vectorize. **The winning representation is the one whose per-vector op is a single instruction** — the law that decided nearly every later call. ([010–012](/experiments/011-asym-lut))

## Making the bits count

One bit/dim is brutal, so don't waste bits. A **random rotation** spreads information evenly across dimensions (free recall). The bigger lever is the **residual**: vectors in a cluster share a center, so their sign bits encode that shared center — useless. Subtract the centroid first and the bits encode each vector's *deviation*. **+3–9 recall points**, free. Toggle it: ([026](/experiments/026-rabitq-rotation), [046](/experiments/046-cell-size-residual))

## Recall is a dial, not a constant

Every experiment trades recall against speed and bytes, so I track recall at each operating point rather than chasing one number. The funnel's recall is set by how wide the shortlist is and how good the codes are — rotation and residual buy recall for free; widening the shortlist buys it for QPS. ([051](/experiments/051-stacked-engine))

## Throughput: tiling

The scan is bandwidth-bound, so reuse each loaded code: compare it against a **tile** of queries while it's hot in cache, before moving on. More comparisons per byte read; the base streams once per tile instead of per query. The tile has to fit in cache (L1/L2), which bounds how big it can get — tile≈8 was the sweet spot here. **+73% QPS**, identical results. ([016](/experiments/016-tiled-binary-scan), [038](/experiments/038-tile-retune))

## The frontier: recall vs QPS

Funnel + rotation + residual + tiling gives a clean frontier. Against Product Quantization (the textbook quantizer, *fewer* bytes), the funnel wins ~7× on QPS at equal recall — lever two again (PQ's op is a gather). ([051](/experiments/051-stacked-engine), [054](/experiments/054-product-quantization))

## Serving without a cliff

A naive one-query-at-a-time server cliffs from ms to *seconds* the moment load exceeds capacity. The **carousel** has queries ride one shared, continuously-cycling scan — every byte read is shared by everyone aboard. Latency stays bounded: ~5 ms light, ~50 ms at the ~940 QPS ceiling, no cliff. ([039–041](/experiments/041-carousel-fanout-frontier))

## Things I tried but don't use

- **Disk-resident vectors** — I measured the economics (codes in RAM, vectors on SSD = 32× less RAM). But warm disk just becomes page-cache-bound, i.e. back to CPU/bandwidth, and right now I'm optimizing for **maximum QPS without a cost constraint** — so the engine stays fully in RAM. Parked, not used. ([045](/experiments/045-disk-resident-economics), [052](/experiments/052-carousel-disk-stack))
- **Scalar PQ / OPQ** (footprint, not speed), **HNSW inside a cell** (loses to a flat 1-bit scan at high-D), **fewer scan bits** (recall collapses for ~8% QPS), **bf16/int8 rerank, register-tiling, prefetch** (neutral-to-negative). All in [/experiments](/experiments).

## The model, and scale

After enough data the engine is predictable: throughput ≈ `1.1e9 · cores / N`, recall a power law in N, shortlist, and bits — separable axes, accurate to ~1 recall point. And it scales: pushed to 100M codes, scan throughput stays flat across the cache→DRAM transition — *compactness is the scaling property*. ([047](/experiments/047-scale-hierarchy-cliff), [049](/experiments/049-roofline-model))

## The surprise

"Popcount always beats gather" held — until I wrote the *SIMD* version of PQ's lookup: 4-bit codes, an in-register `pshufb` table (safe portable-SIMD Rust), 16 lookups per instruction. At 8 bytes/vector it stays in cache and runs **2–9× faster than an untiled popcount scan** (the green bar above). Gather isn't dead — it had to be vectorized *and* tiny. Whether it beats the *tiled* funnel end-to-end is the next thing I'd chase. ([059](/experiments/059-simd-adc-pq4))

## A note on the hardware

This needs AVX-512 (`VPOPCNTDQ`, `pshufb`). I prototyped on Lightsail, then moved to an EC2 c8i (Granite Rapids — closer to metal, real AVX-512) but kept the instance modest to not burn money. Everything here is one 8-core box; scaling past ~940 QPS is "add cores/shards," which the model predicts.

## The best, in one line

A **1-bit binary funnel** — rotation + residual to make the bits count, tiling for throughput, a **carousel** so latency never cliffs — does **~0.995 recall at ~960 QPS and 8 ms** on one 8-core box, in 128 MB. The deepest lesson: on a CPU, the representation that wins is the one whose per-vector operation is a single SIMD instruction.
